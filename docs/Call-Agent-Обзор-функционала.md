# Call-Agent — полный обзор функционала

**Call-Agent (ОКК)** — система AI-контроля качества коммуникаций отдела продаж. Автоматически забирает из CRM звонки, чаты, письма и встречи менеджеров, расшифровывает их, анализирует нейросетью по вашему скрипту и чек-листу, и показывает руководителю объективную картину: кто как продаёт, где теряются сделки, что улучшить.

Работает поверх **Bitrix24** (поддержка amoCRM и других — на подключении). Менеджеры ничего не делают руками — анализ идёт фоном.

---

## 1. Как это работает (поток за 4 шага)

```
1. ЗАБОР          2. РАСШИФРОВКА      3. AI-АНАЛИЗ          4. РЕЗУЛЬТАТ
Звонки/чаты/   →  Аудио → текст   →  Claude/GPT по      →  Дашборд, оценки,
письма/встречи    (Whisper)          скрипту+чек-листу     отчёты, запись в CRM
из Bitrix24                          вашей компании
```

1. **Забор данных** — система сама, без участия менеджеров, тянет из Bitrix24 новые звонки (телефония), переписки (WhatsApp/Telegram/Open Lines), письма и завершённые встречи. Каждые несколько минут.
2. **Расшифровка** — аудиозаписи звонков превращаются в текст (распознавание речи Whisper), с разделением реплик «менеджер / клиент».
3. **AI-анализ** — нейросеть читает разговор и оценивает его по **вашему** скрипту продаж и чек-листу качества: выставляет оценку, определяет тональность, находит возражения, темы, следующий шаг, даёт менеджеру персональные подсказки.
4. **Результат** — всё стекается в наглядный дашборд для руководителя; по желанию краткий итог анализа автоматически пишется комментарием в карточку сделки Bitrix, а сводные отчёты рассылаются в мессенджер.

---

## 2. Источники данных (что анализируем)

| Тип | Откуда | Как анализируется |
|---|---|---|
| **Звонки** | Телефония Bitrix24 (вкл. внешние АТС — Телфин и др.) | Аудио → расшифровка → анализ |
| **Встречи** | CRM-активность «Встреча» (итог, который менеджер записал) | Текст итога → анализ |
| **Чаты** | Open Lines: WhatsApp, Telegram, ВК и др. | Переписка → анализ |
| **Письма** | Email-активности Bitrix | Текст письма → анализ |
| **Ручная загрузка** | Загрузить аудио/текст вручную (Zoom, диктофон, любой файл) | Расшифровка/текст → анализ |

Звонки **без записи** (внутренние, пропущенные) учитываются в статистике, но не тратят ресурсы на анализ — помечаются отдельным статусом.

---

## 3. Что AI извлекает из каждого разговора

По каждому звонку/встрече система выдаёт:

- **Оценка менеджера** (0–10) — насколько качественно проведён разговор
- **Выполнение чек-листа** (%) — по пунктам вашего стандарта (приветствие, выявление потребности, презентация выгод, отработка возражений, договорённость о следующем шаге и т.д.) с оценкой каждого пункта
- **Тональность** разговора (позитив / нейтрал / негатив)
- **Краткое содержание** — суть разговора в 1–2 предложениях
- **Что хочет клиент** — выявленная потребность
- **Возражения клиента** — список («дорого», «работаем с другими», «нужно посоветоваться»…)
- **Темы разговора** — цена, сроки, доставка, гарантия…
- **Следующий шаг** — что менеджер пообещал/должен сделать (с датой)
- **Этап сделки** — на какой стадии находится клиент
- **Подсказки менеджеру (коучинг)** — персональные рекомендации что улучшить
- **Полная расшифровка** с разделением реплик менеджер/клиент
- **Имя/компания клиента** — распознаётся из разговора

## 4. Модули интерфейса

### Дашборд руководителя
Главный экран. Гибкие фильтры (сегодня / вчера / неделя / месяц / произвольный период; по менеджеру; только сделки с CRM-привязкой).
- **KPI-плитки:** всего звонков, проанализировано, в обработке, без записи, ошибки; средняя оценка, средний чек-лист, средняя длительность, входящие/исходящие
- **Таблица менеджеров** — детальная статистика по каждому: звонки, минуты, % контактов (дозвонов), пропущенные, средняя оценка, чек-лист, тональность
- **Динамика за 14 дней** — график активности и настроения
- **Распределение по продуктам** (направлениям) — МП / МК и т.д.
- **Топ возражений** клиентов и **топ тем**
- **Слабые места в скрипте** — какие пункты чек-листа команда выполняет хуже всего (зоны роста)
- **«Поделиться»** — публичная ссылка на дашборд (read-only) для собственника без логина

### Звонки (список взаимодействий)
Все звонки/чаты/письма/встречи с фильтрами (тип, тональность, статус, длительность, менеджер, период, поиск по тексту расшифровки). Иконка типа у каждой записи. Клик → карточка.

### Карточка звонка
Полная картина одного разговора: данные (менеджер, клиент, направление, длительность), плеер записи, **связка с CRM** (сделка/лид/контакт + ссылка в Bitrix), полная расшифровка-диалог, весь AI-анализ (оценка, чек-лист по пунктам, возражения, следующий шаг, коучинг). Кнопки: **«Провести более глубокий анализ»**, **«Переоценить»**, **«Перезапустить обработку»**.

### Заказчики (профиль 360)
Все клиенты, агрегированные по телефону. Профиль клиента: вся история касаний (звонки/чаты/письма/встречи в одной ленте), динамика тональности, **«оборванные нити»** — клиенты, с которыми давно не было контакта, хотя сделка жива.

### Сравнение с CRM-карточкой («Расхождения»)
Система сверяет, что обсуждалось в разговоре, с тем, что записано в карточке сделки Bitrix, и подсвечивает **расхождения**: например, в разговоре клиент согласовал сумму 250 000, а в сделке стоит 150 000; или изменился контакт/срок/этап. Руководитель видит, где менеджеры не довносят данные в CRM. Можно принять/отклонить, ИИ может внести правку сам.

### Отчёты
Формирование сводных отчётов за период:
- **По менеджеру** (персональный) и **Общий по команде**
- **Выбор получателя** — отправить отчёт в личку конкретному человеку (РОПу) или в **групповой чат Bitrix**, независимо от того, про кого отчёт
- **Авто-расписание** — отчёты уходят автоматически: ежедневно/еженедельно в заданное время (например, каждое утро в 9:00 отчёт за вчера РОПу в чат). Несколько расписаний одновременно
- Отчёт содержит дату, цифры, оценки по менеджерам, зоны роста, частые возражения, ссылку на дашборд

### Лидерборд (геймификация)
Рейтинг менеджеров, достижения (ачивки), серии (streaks) — мотивационный слой для команды.

### Кабинет менеджера
Каждый менеджер видит **только свои** звонки и метрики, свои зоны роста и ленту персональных подсказок от ИИ — инструмент самообучения.

### Очередь и CRM-журнал
- **Очередь** — что сейчас обрабатывается (прозрачность пайплайна)
- **CRM-журнал** — история записей, отправленных в Bitrix

## 5. Скрипты продаж и чек-листы (настраивается под вас)

- **Несколько скриптов** под разные продукты/направления (например «Модульные павильоны» и «Металлоконструкции») — ИИ сам определяет по разговору, какой это продукт, и оценивает по нужному скрипту
- **Чек-лист качества** — пункты с весами и описанием; загружается из **.docx / Excel** или редактируется в интерфейсе. Блоки/разделы
- **Ключевые фразы** — подсказки ИИ для распознавания типа разговора
- **Закрепление продукта за менеджером** — если менеджер ведёт в основном один продукт, его можно закрепить: ИИ будет считать этот продукт приоритетным (но всё равно проверит по содержанию)

---

## 6. Запись анализа обратно в Bitrix (CRM-write)

Краткий итог анализа (содержание + следующий шаг) автоматически пишется **комментарием в карточку сделки/лида** Bitrix — менеджеры и РОП видят выжимку прямо в CRM, не заходя в Call-Agent.
- **Точечное включение** — можно выбрать, для каких менеджеров переносить комментарии (например, сначала на одном, потом на всех)
- **Предохранитель DRY_RUN** — два независимых рубильника: отдельно для комментариев в CRM, отдельно для отчётов в мессенджер. Можно тестировать одно, не трогая другое
- Описание активности в комментарий не тянется — только суть

---

## 7. Интеграции с Bitrix24

- **Телефония** — звонки + записи (включая внешние АТС через CRM-активности)
- **CRM** — привязка к сделкам/лидам/контактам, имена и ссылки на карточки, запись комментариев, сверка расхождений
- **Мессенджер** — рассылка отчётов в личку и групповые чаты
- **Активности** — письма, чаты Open Lines, встречи
- Подключение — через входящий вебхук Bitrix (telephony, crm, user, im, imbot, disk, mail)

---

## 8. Гибкость AI и контроль расходов

- **Выбор модели анализа** — на тенант можно выбрать модель (баланс качество/цена: от экономной Haiku/GPT-4o-mini до премиальной Sonnet/GPT-4o)
- **Бюджет-гард** — лимит расхода токенов/минут на период, чтобы расходы были предсказуемы
- **Авто-повтор** при временных сбоях провайдера; **баннер-предупреждение**, если у AI-провайдера закончилась квота (анализ не «зависает» молча)
- **Глоссарий** — правильные написания названий компании/брендов, чтобы ИИ не путал

---

## 9. Роли и доступ

- **Владелец / Администратор** — всё
- **РОП (руководитель)** — командные дашборды, отчёты, расхождения
- **Менеджер** — только свой кабинет и свои звонки
- **Демо** — публичный просмотр витрины (read-only)
- Мультиаккаунт, изоляция данных по компании (тенанту)

---

## 10. Демо-режим (для показа клиентам)

Публичная ссылка **`/call-agent/demo`** — открывается без пароля, показывает полностью заполненную витрину на вымышленной компании «ООО Ромашка»: дашборд, звонки с анализом, заказчиков, скрипты, отчёты, расхождения. **Только просмотр** — изменить ничего нельзя (защита на уровне сервера). Идеально для демонстрации возможностей потенциальному клиенту.

---

## 11. Онбординг новой компании

Публичная форма-опросник собирает всё, что нужно для подключения новой компании с нуля: данные компании, доступ к Bitrix (вебхук), тип телефонии, список менеджеров и РОПа, скрипт/чек-лист, продукты, настройки анализа, параметры отчётов. Заявки видит администратор.

---

## 12. Надёжность и безопасность

- Авто-импорт и авто-повтор упавших задач (сетевые сбои, перегрузка провайдера)
- Обход гео-блокировки AI-провайдеров для работы из РФ
- Защита от утечки данных между компаниями (изоляция по тенанту)
- Предохранители DRY_RUN — ничего не уходит наружу, пока вы не разрешите
- Демо строго read-only на уровне сервера

---

## Кратко: зачем это бизнесу

- **Руководитель** видит реальную картину продаж без ручного прослушивания звонков
- **100% разговоров** под контролем качества (а не выборка)
- **Объективная оценка** менеджеров по единому стандарту
- **Зоны роста** команды и каждого менеджера — на ладони
- **Данные в CRM** перестают теряться (сверка расхождений)
- **Автоматические отчёты** — РОП получает сводку каждое утро, не запрашивая
